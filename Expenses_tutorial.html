1) Expenses Overview

Menu area: Expenses (module) → Expenses list

The Expenses module records costs incurred in operating or maintaining a property (e.g. plumbing repairs, electricity bills, painting, cleaning, security services). Expenses can be linked to:

  • A Property (required)
  • An Apartment/Unit (optional, where the expense is unit-specific)
  • A Major Expense Category (required; e.g. Utilities, Repairs, Administration)
  • A Minor Expense Category (required; e.g. Electricity Bill, Toilet Repairs, Stationery)
  • A Payee (optional/required depending on configuration)
  • Attachments (optional but recommended; receipts, invoices, work completion evidence)

Prerequisites (recommended setup order)

  1. Create Properties and Apartments/Units (if unit-level allocation is required).
  2. Configure Major Expense Categories and Minor Expense Categories.
  3. Configure Payees (common suppliers/contractors) for consistent reporting.
Best practice: Always enter an expense with a clear Narration and correct Date Incurred. Attach evidence (receipt/invoice/photo) to strengthen audit trail and owner reporting.

2) Expenses List

Screen: Expenses list (with SEARCH and summary totals)

The expenses list shows recorded expense entries. Typical summary cards include:

  • Total Items: count of expense records.
  • Total Expenses: sum of expenses (based on active filters/date scope).
  • Expenses this month: expenses incurred within the current month.

Use SEARCH to quickly locate expenses by narration, property, apartment/unit, or payee.

3) Add Expense

Action: Expenses list → Tap + to add a new expense

Typical fields

  • Select Property (required)
  • Select Apartment/Unit (optional)
  • Major Expense Category and Minor Expense Category
  • Select Payee (where applicable)
  • Date Incurred
  • Expense Amount
  • Narration (required; describe the expense clearly)
Tip: Consistent category and payee usage makes reporting, audits, and analytics cleaner.

4) Add Images / Attachments (Supporting Documents)

Where: Add Expense screen → Add Images / Attachments

You can attach supporting documents to an expense record. Attachments may include:

  • Photos (take a photo or choose from gallery)
  • PDF documents (supplier invoice, receipt, quotation)
  • Word documents (signed job note, supplier quotation, approvals)
Operational value: Attachments create a complete “expense pack” for owners, internal review, and audit compliance.

5) Expense Actions (Long-Press Context Menu)

Action: Expenses list → Long-press an expense card

Common actions include:

  • Edit: update expense details.
  • Delete: remove expense record (use for duplicates/incorrect entries).
  • View Attachments: review evidence linked to expense.
  • Export Images to PDF: compile evidence into a PDF for sharing.

6) Attachments & PDF Export

Menu path: Expenses list → Long-press expense → View Attachments / Export Images to PDF

Why export to PDF?

  • Send a single consolidated document to owners or internal teams.
  • Attach evidence to emails and owner reports.
  • Preserve an audit trail (expense + evidence together).
Recommendation: Export after attaching evidence so each expense has a complete supporting pack.

7) Bulk Import Expenses (Initial Setup)

Screen: Expenses list → Template (small +) and CSV Import buttons

Bulk import is recommended where a large number of expenses must be migrated during initial setup.

  1. Tap the template button (small “+”) to generate/download the template.
  2. Fill the template in Excel (one row per expense).
  3. Save as CSV.
  4. Import using the CSV button.
Import hygiene: Do not rename headers. Ensure required fields are filled. Validate dates and amounts before importing.

8) Inventory Management — Navigation

Menu path: Main menu (☰) → Inventory Items

The Inventory Items module tracks physical assets used in property operations (e.g., meters, furniture, appliances). It supports registering items, tracking value, assigning items to locations, and disposing items that are no longer in service.

9) Inventory Items — Landing Page & Long-Press Actions

Screen: Inventory Items list (with SEARCH and summary totals)

Context menu actions (long-press on an item)

Menu item What it does When to use it / rules
Edit Item Update item details (name, number, category, brand, condition, purchase info). Use to correct data or update item attributes.
Delete Item Remove item record. Use for wrong/duplicate entries. For items leaving service, prefer Create Item Disposal to keep history.
Assign Item Assign item to a location. Use when installing/issuing to a unit/location.
View Item Assignment View assignment history/current placement. Use to confirm where the item is placed and review movement history.
Create Item Disposal Record item disposal and mark item inactive. Use when item is out of service (sold/scrapped/lost/donated/stolen/transferred).
View Item Disposal View disposal record(s) for that item. Use to confirm disposal details and audit trail.
Key control: Assignment answers “Where is the item?” Disposal answers “Is the item still usable/active?” A disposed item should not be assigned again unless disposal is reversed and the item is reactivated (where supported).

11) Assign Item

Action: Inventory Items list → Long-press item → Assign Item

Use this screen to assign an item to a location (e.g., Kitchen → Main Kitchen, or a unit-specific location as configured).

Important rule: An item can only be assigned to another location when the current assignment is ended.

12) Assignment Actions (Edit, Delete, End Assignment)

Action: View Item Assignment → Long-press an assignment record

Ending the assignment closes the current placement of an item. The item can only be re-assigned after the active assignment is ended.

  • Edit: correct assignment data (location/date/notes).
  • Delete: remove wrong/duplicate assignment records.
  • End Assignment: close active assignment so the item can move.

Back to top

13) Item Disposal (Create and View Disposal)

Action: Inventory Items list → Long-press item → Create Item Disposal / View Item Disposal

When to use disposal

Disposal is used when an item is no longer in service or no longer available, such as: SOLD, SCRAPPED, LOST, DONATED, STOLEN, TRANSFERRED.

Recommended control: End active assignment first before disposal to avoid an item appearing as both “assigned” and “disposed”. After disposal, the item is marked inactive and should not be assigned again unless disposal is reversed and item is reactivated (if supported).
