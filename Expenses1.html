Landlord App — Expenses & Inventory Guide

This document explains how to capture property expenses, classify them (major/minor categories), track payees, add supporting attachments (photos, PDF, Word), export attachments to PDF, and bulk-import expense data using templates. It also covers Inventory Management: navigation, inventory landing page, add inventory items, assignment workflows, ending assignments, item disposal workflows, and bulk import of inventory records.

1) Expenses Overview

Menu area: Expenses (module) → Expenses list

The Expenses module records costs incurred in operating or maintaining a property (e.g., plumbing repairs, electricity bills, painting, cleaning, security services). Expenses can be linked to:

  • A Property (required)
  • An Apartment/Unit (optional, where the expense is unit-specific)
  • A Major Expense Category (required; e.g., Utilities, Repairs, Administration)
  • A Minor Expense Category (required; e.g., Electricity Bill, Toilet Repairs, Stationery)
  • A Payee (optional/required depending on your configuration; e.g., Plumber, Supplier, Company)
  • Attachments (optional but recommended; receipts, invoices, work completion evidence)

Prerequisites (recommended setup order)

  1. Create Properties and Apartments/Units (if you want unit-level allocation).
  2. Configure Major Expense Categories and Minor Expense Categories.
  3. Configure Payees (common suppliers/contractors) for consistent reporting.
Best practice: Always enter an expense with a clear Narration (what the cost was for) and a correct Date Incurred. Attach supporting documents (receipt/invoice/photo) to strengthen audit trail and owner reporting.

8) Inventory Management — Navigation

Menu path: Main menu (☰) → Inventory Items

The Inventory Items module is used to track physical assets owned/used in property operations, such as meters, furniture, appliances, tools, or equipment. It supports registering items, tracking their value, assigning items to a location, and disposing items that are no longer in service.

9) Inventory Items — Landing Page

Screen: Inventory Items list (with SEARCH and summary totals)

The inventory landing page shows a list of items recorded in the system. At the top, the app typically shows summary cards such as: Total Items, Active Items, and Total Value.

Item-level actions (long-press)

Long-press an item card to open the context menu. Typical actions include: Edit Item, Delete Item, Assign Item, View Item Assignment, Create Item Disposal, and View Item Disposal.

Menu item What it does When to use it / important rules
Edit Item Updates item details (name, number, purchase value/date, category, brand, condition). Use to correct mistakes or update item attributes.
Delete Item Removes the item record. Use only for wrong/duplicate items created by mistake. For items that are no longer usable, prefer Create Item Disposal to keep history and audit trail.
Assign Item Assigns the item to a location (unit or property area). Use when an item is installed/issued to a location.
View Item Assignment Shows assignment history and current active assignment. Use to confirm where an item is currently placed and review movement history.
Create Item Disposal Records that an item has been disposed (out of service) and marks it inactive. Use when an item is no longer available for use (e.g., sold, scrapped, lost, donated, stolen, transferred). In most implementations, each item should have only one disposal record (to avoid conflicting status).
View Item Disposal Shows disposal records (history) for that item. Use to confirm disposal details and audit the reason/date for deactivation.
Key concept: Assignment answers “Where is the item placed?” while Disposal answers “Is the item still active/usable?” A disposed item should not be re-assigned until the disposal entry is reversed/removed and the item is reactivated (if your system supports reactivation).

10) Add Inventory Item

Action: Inventory Items list → Tap + to add a new item

Use the add inventory screen to register an item under a property, capture its identifying details (name and item number), and categorize it (condition, brand, category). Purchase date is optional.

Typical fields on Add Inventory screen

  • Select Property
  • Item Name
  • Item Number (serial/meter/asset ID)
  • Purchase Date (optional)
  • Purchase Value
  • Item Condition (e.g., New)
  • Brand (optional)
  • Category
Tip: Item Number should be unique where possible. This supports inspections, handovers, and audit reconciliation.

11) Assign Item (Inventory Assignment Screen)

Action: Inventory Items list → Long-press item → Assign Item

The Assign Item screen is used to assign an inventory item to a physical location. This can be a unit location (e.g., Apartment 001) or another defined place within the property (e.g., Main Kitchen), depending on the configured Location Types.

Fields on the Assign Item screen

Field What to select / enter Notes
Item Condition Select the condition at the time of assignment. Useful for tracking deterioration over time.
Location Type Select the location type (e.g., Kitchen). Location Types standardize where items can be assigned.
Location Select the specific location (e.g., Main Kitchen). Represents the exact assigned place.
Assignment Date Tap to select date. Records when the item started being used in that location.
Notes (optional) Add any notes about the assignment. Example: “Installed by plumber”, “Newly issued”.

Assigning workflow

  1. Choose Location Type and Location.
  2. Select Assignment Date.
  3. Optionally add Notes.
  4. Tap Assign Item to save the assignment.
Important rule: An item can only have one active assignment at a time. To assign the item to a new location, you must first End Assignment on the current assignment.

12) View Item Assignment (Assignment Landing Page)

Action: Inventory Items list → Long-press item → View Item Assignment

The View Item Assignment screen shows assignment records for an item. Each record typically includes:

  • Item Name
  • Assignment Date
  • Item Condition
  • Place Assigned (location code/name)
  • Assignment End Date:
    • Active indicates the assignment is still open.
    • A date indicates the assignment has been ended/closed.

Long-press an assignment record on this screen to open additional actions such as Edit, Delete, and End Assignment.

13) Assignment Actions (Edit, Delete, End Assignment)

Action: View Item Assignment → Long-press an assignment record
Action What it does When to use it
Edit Updates assignment details (location type/location, assignment date, notes, condition). Use to correct mistakes (wrong location selected, wrong date, missing notes).
Delete Removes an assignment record. Use only for wrong/duplicate assignment entries. For real movement history, prefer ending assignments.
End Assignment Closes an active assignment by recording an end date (assignment becomes inactive). Use when an item is removed from that location (transfer, replacement, disposal, relocation). You must end the active assignment before assigning the same item to a new location.
Operational rule: An item cannot be assigned to another location while the current assignment is still Active. End the current assignment first, then create a new assignment for the new location.

14) Item Disposal (Taking Items Out of Service)

Action: Inventory Items list → Long-press item → Create Item Disposal / View Item Disposal

Item Disposal is used to record that an item is no longer in service (no longer usable or no longer owned/available), while keeping a clean audit trail for operations and reporting. Typical disposal types include: SOLD, SCRAPPED, LOST, DONATED, STOLEN, and TRANSFERRED.

Recommended enforcement: One disposal entry per item (unique disposal per item) to ensure the item has only one final “out of service” state. If you need to reverse a disposal, delete the disposal entry and reactivate the item (if your app supports this workflow).

Key rules (to prevent inconsistent status)

  • If an item has an active assignment, you should end the assignment first before disposal (so the item is not shown as “actively assigned” while disposed).
  • After disposal, the item should be marked Inactive and excluded from Active Items counts by default.
  • A disposed item should not be assigned to a new location until it is reactivated (if reactivation is allowed).

15) Create Item Disposal

Action: Inventory Items list → Long-press item → Create Item Disposal

Use this action to capture the disposal transaction/event for a specific item. The disposal record should capture at least:

  • Item (the selected item_id from the list)
  • Disposal Date (when the item left service)
  • Disposal Type (e.g., Sold, Scrapped, Lost, Donated, Stolen, Transferred)
  • Notes/Remarks (optional but recommended: buyer, police ref, transfer details, etc.)

What happens after saving a disposal record

  • The item is marked Inactive (example: date_made_inactive = disposal date).
  • The item will normally not appear under “Active Items” totals.
  • The disposal entry becomes part of the audit trail and can be viewed under View Item Disposal.
Implementation note: If you enforce a unique disposal per item, the system should prevent creating a second disposal entry for the same item. In that case, direct the user to View Item Disposal and optionally allow editing/removing the disposal entry (with proper permissions).

16) View Item Disposal (Disposal History)

Action: Inventory Items list → Long-press item → View Item Disposal

This screen shows disposal records for the selected item (or the single disposal record, if you enforce “one disposal per item”). It helps users confirm when and why an item was taken out of service.

Typical actions on disposal records

Action What it does When to use it / important rules
Delete Disposal (Undo) Deletes the disposal entry and reactivates the item (if your workflow supports undo). Use if disposal was recorded by mistake. After deletion, set the item back to active (clear inactive date), then the item can be assigned again if needed.
Operational control: If you support “Undo Disposal” by deleting the disposal record, ensure the item is reactivated and ensure any assignment rules remain valid (for example, do not automatically create a new assignment; user should assign explicitly).

17) Bulk Import Inventory Items (Initial Setup)

Screen: Inventory Items list → Template (small +) and CSV Import buttons

Inventory records can be bulk imported into the app, and it follows the same pattern used in other modules: generate a template → fill it in Excel → save as CSV → import using the CSV button.

How it works (Template → Fill in Excel → Save CSV → Import)

  1. Tap the template button (small “+”) to generate/download the inventory import template.
  2. Fill the template in Excel using the provided columns only (one row per item).
  3. If the template includes a date field (e.g., Purchase Date), ensure it is formatted as yyyy-mm-dd before saving CSV.
  4. Save/export as CSV.
  5. Tap the CSV button and select the saved CSV file to import.
Important rules for successful import:
  • Do not change template headers or column order unless the app explicitly allows it.
  • Ensure required fields are provided for every row (especially Property and Item Name/Number as required by the template).
  • After import, verify a sample of items and confirm totals (Total Items / Total Value).
