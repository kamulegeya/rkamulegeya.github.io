1) Tenants Landing Page (Add Tenants tab)

Menu area: Tenants → Add Tenants (tab)

When you open the Tenants module, the landing page shows a list of saved tenants. Each record displays key fields such as Name and Tel, and may show other details (TIN, NIN, Email, Next of Kin) where available.

Common actions

  • Search: quickly locate a tenant (by name/phone depending on your app search behavior).
  • Edit (pencil icon): update tenant details.
  • Delete (X icon): remove a tenant record.
  • Add: create a new tenant.
Recommendation: Keep tenant records complete (phone number, IDs, and email where applicable) to support accurate billing, compliance, and communication. For large initial setup, consider Bulk Import Tenants using the template and CSV import workflow.

2) Create Tenant

Menu area: Tenants → Add Tenants → Add Tenant

Steps

  1. Open the Tenants module.
  2. Stay on the Add Tenants tab.
  3. Tap + (or Add Tenant) to open the tenant form.
  4. Select a Tenant Type, then complete the fields shown for that type.
  5. Tap Add Tenant to save.

Tenant fields (common)

Field (as shown in the app) What to enter Notes
Tenant Type (spinner) Select tenant type (e.g., Individual, Government). Controls which detail section appears below.
Full Name Tenant name (person or organization name). Use the legal name for proper records and billing.
Telephone Number Tenant phone number. Recommended for rent follow-up and communication.
Tenant TIN Tenant tax ID / TIN (if applicable). Often required for corporate/government tenants depending on compliance needs.

Individual Tenant Details (when Tenant Type = Individual)

Field What to enter Notes
Email Address Tenant email. Useful for sending statements/invoices where applicable.
National ID (NIN) National Identification Number. Use official ID format.
Next of Kin Next of kin name. Recommended for emergency/reference details.
Next of Kin Tel. Number Next of kin phone number. Ensure it is reachable.

Corporate / Organization Details (when Tenant Type is Government / Organization)

Field What to enter Notes
Contact Person Main contact name at the organization. Used for official communication and follow-up.
Contact Person Telephone Contact person phone number. Use a direct/active line where possible.
Contact Person Email Contact person email. Recommended for official notices and billing communication.

3) Bulk Import Tenants (Initial Setup)

Menu area: Tenants → Add Tenants (tab)

For big landlords and agencies, tenant onboarding can be accelerated by capturing tenant data in an Excel template and bulk-importing it into the app. The template is generated from the Tenants screen, completed on a laptop/desktop, saved as CSV, then imported back into the app.

When to use bulk import

  • Initial onboarding where you have a substantial number of tenants to capture at once.
  • You already have tenant records in a spreadsheet and need a quick migration approach.

How it works (Template → Fill in Excel → Save CSV → Import)

  1. On Tenants → Add Tenants, tap the template button (the button with a small “+” icon). The app generates/downloads a tenant import template.
  2. Open the template on a computer (Excel or another spreadsheet tool). Fill in the tenant records using the provided columns only.
  3. Save/export the completed sheet as a CSV file (Comma Separated Values).
  4. Back on the Add Tenants screen, tap the CSV Import button (the button labeled CSV) and select the saved CSV file to import.
Important rules for successful import:
  • Do not change the template headers (column names) or their order unless the app explicitly allows it.
  • Complete all required fields for every tenant row (especially tenant name and telephone).
  • Use consistent formatting for phone numbers and IDs to minimize duplicates and failed imports.
  • Prefer saving CSV as UTF-8 where names may include special characters.
  • After import, search a few tenants and confirm key details (type, telephone, and ID fields) are correct.

4) Create Tenancy (Assign Tenant to an Apartment/Unit)

Menu area: Tenants → Add Tenancy (tab)

A Tenancy links a tenant to an apartment/unit, defines the tenancy term rules, sets the billing currency, and captures rent amount and dates. It can also include Recurring Charges (extra charges billed regularly).

Fields on the Add Tenancy screen

Field What to select / enter Notes
Select a tenant (spinner) Choose the tenant to occupy the unit. Tenant must be created first.
Select an apartment (spinner) Choose the apartment/unit to be occupied. Apartment/unit must be created first.
Currency (spinner) Select the billing currency for this tenancy. This currency is used for the rent amount and recurring charges.
Terms (spinner) Select a tenancy term (e.g., Default Terms). Terms define grace days, deposit, escalation rules, and late charges (where configured).
Start date Enter/select the tenancy start date. Used to determine when billing/occupancy begins.
End date Enter/select an end date (if applicable). Optional for open-ended tenancies; required for fixed-term leases.
Monthly rent amount Enter the rent amount for the tenancy. Use numbers only where possible (no currency symbols).
Recurring Charges Tap + Add Recurring Item to add repeating charges. Examples: cleaning, security, maintenance. Behavior depends on your billing rules.

Save Tenancy

  1. Confirm Tenant, Apartment, Currency, and Terms are correct.
  2. Enter Start date, optional End date, and Monthly rent amount.
  3. Add recurring charges if needed.
  4. Tap Save Tenancy.

5) Bulk Import Tenancies (Initial Setup)

Menu area: Tenants → Add Tenancy (tab)

For big landlords and agencies, you can also bulk-import tenancy records for initial setup. This is useful when you already have an occupancy list (tenant-to-unit assignments) and want to onboard it quickly.

When to use bulk import

  • Initial onboarding where many units already have sitting tenants and you want to load assignments fast.
  • Migrating tenancy history/starting positions from an existing spreadsheet system.

How it works (Template → Fill in Excel → Save CSV → Import)

  1. On Tenants → Add Tenancy, tap the template button (the button with a small “+” icon). The app generates/downloads a tenancy import template.
  2. Open the template on a computer (Excel or another spreadsheet tool) and fill the tenancy rows using the provided columns only.
  3. Save/export the completed sheet as a CSV file.
  4. Back on the Add Tenancy screen, tap the CSV Import button (the button labeled CSV) and select the CSV file to import.
Critical prerequisite: Tenancies reference other master data. Ensure the following exist before importing tenancies:
  • Tenants are already created/imported.
  • Apartments/Units are already created/imported.
  • Tenancy Terms are configured.
  • Currencies required by the tenancies are configured.

Start Date and End Date import rules (very important)

Date format is strict: In the tenancy import CSV, Start Date and End Date must be supplied in the exact format yyyy-mm-dd.

Examples: 2025-01-12, 2025-11-30
Not allowed / risky: 12/01/2025, 01/12/2025, 12-01-2025, 2025/01/12, or any value that includes time such as 2025-01-12 00:00:00.

How to format dates correctly in Excel before saving CSV

  1. Select the Start Date and End Date columns in the template.
  2. Right-click → Format Cells → Custom → set to yyyy-mm-dd.
  3. Confirm every row displays dates like 2025-01-12 (4-digit year, 2-digit month, 2-digit day).
  4. For maximum reliability, you may force Excel to output the exact text using a helper formula:
    • In a helper column use: =TEXT(A2,"yyyy-mm-dd") (adjust A2 to your date cell).
    • Copy the results → Paste Special → Values back into the Start/End Date columns.
  5. Save the file as CSV and import using the app CSV button.

Final verification before import (recommended)

  • Open the saved CSV in a plain text editor (e.g., Notepad) and confirm dates still appear as yyyy-mm-dd.
  • If your CSV shows slashes, ambiguous formats, or time, correct the Excel formatting and re-save.
End Date handling: If your tenancy template allows open-ended tenancies, leave End Date blank (empty cell) for ongoing leases. Avoid using N/A, 0, or other placeholders unless the template explicitly requires them.
Important rules for successful import:
  • Do not change template headers or column order unless the app explicitly allows it.
  • Use valid references exactly as expected by the template (for example: exact tenant identifiers/names and exact unit identifiers/names as used in the app lists).
  • Ensure Start Date is not after End Date.
  • Confirm rent amounts and currencies are correct; incorrect currency selection will cause downstream billing errors.
  • After import, review a sample of tenancies to confirm tenant-to-unit mapping, terms, start dates, and rent amounts.

6) Tenancy List & Context Menu Actions

Menu area: Tenants → Add Tenancy (tab) → Tenancy list

The tenancy list displays saved tenancies and key details such as Tenant, Apartment, Billing Currency, Start/End dates, Amount, Status, and Tenancy Terms.

Opening the tenancy action menu (long-press)

  • Long-press on a tenancy card to open the tenancy context menu.
  • Use SEARCH to locate a tenancy quickly.

Context menu items (what each one does)

Menu item What it does When to use it
Edit Updates tenancy details. Correct unit selection, currency, terms, dates, rent amount, or recurring items.
Delete Removes the tenancy record. Use only for incorrect/duplicate tenancies; prefer Terminate for real historical tenancies.
Terminate Ends an ongoing tenancy and records it as terminated. Use when a tenant moves out or the lease ends early.
Create Statement Generates a statement for the tenancy. Use for payment history and balances (based on your statement rules).
Add Documents Attach documents to the tenancy. IDs, agreements, move-in forms, correspondence, proof of payment.
View Documents Opens the document list for the tenancy. Review previously attached documents.
Create Tenancy Agreement Generates a tenancy agreement document. Ensure tenant/unit/dates/rent/terms are correct before generating.
Renew Tenancy Creates a new tenancy period after the current one ends. Use for fixed-term renewals (new lease period).
Extend Tenancy Extends the current tenancy period (typically end date adjustment). Use when continuing the same tenancy record without creating a new one.
Good practice: Use Terminate for real tenancies that ended, and Delete only for mistakes. Use Renew for a new lease period, and Extend when only adjusting the end date.

7) Security Deposit Fee (Manage Deposits)

Screen: Manage Security Deposit Fees

The Security Deposit Fee module is used to record deposit amounts received from tenants, track whether they have been refunded, and produce receipts where needed.

What you see on the Manage Security Deposit screen

  • Records: number of deposit entries saved.
  • Total Deposits: total value of deposits recorded.
  • Total Refunded: total value of deposits marked as refunded.
  • A list of deposit entries showing fields like:
    • Tenant Name
    • Date Received
    • Fee Amount
    • Description
    • Date Refunded (or “Not Yet Refunded”)
    • Amount Refunded (or “Not Yet Refunded”)
    • Reason Refunded (or “Not Yet Refunded”)
  • SEARCH lets you find a deposit record quickly.

Add a new Security Deposit Fee

Action: Tap the + button → Add new Security Deposit Fee
  1. Tap + to open Add new Security Deposit Fee.
  2. Select the Tenancy the deposit belongs to.
  3. Select the Currency for the deposit.
  4. Select the Date Received (the screen indicates “double click to select a date”).
  5. Enter the Fee Amount.
  6. If required, enter the Exchange Rate.
  7. Enter a Description (e.g., “Security fees”).
  8. Tap Add Security Deposit Fee.
Field (as shown) What to enter / select Notes
Select Tenancy (spinner) Select the tenancy for the tenant/unit. Deposit should be tied to the correct tenancy for accurate records.
Currency (spinner) Select the deposit currency. Use the tenancy currency where applicable.
Date Received Select the date the deposit was received. The screen indicates “double click to select a date”.
Enter fee amount Deposit amount received. Use numbers only where possible.
Enter exchange rate Exchange rate (default is 1). The default value is 1 (meaning no conversion). Keep it as 1 when the deposit currency is already your reporting/home currency. Only change it when you record a deposit in a different currency and you need conversion for reporting.
Enter Description Short reason/label for the deposit. Example: “Security deposit”, “Security fees”.

Deposit context menu (long-press)

How to open actions: Long-press a deposit record to show available actions.

Context menu items (what each one does)

Menu item What it does Typical use
Refund Security Deposit Marks the deposit as refunded and records refund details. Use when returning the deposit to the tenant. Typically captures items such as refund date, amount refunded, and reason refunded (based on your refund form).
Delete Removes the deposit record. Use only for wrong/duplicate entries. For real deposits, prefer refunding instead of deleting.
Create PDF Receipt Generates a PDF receipt for the deposit. Use when you need a printable or shareable receipt for the tenant/records.
Good practice: For accuracy and audit trail, avoid deleting real deposits. Use Refund Security Deposit when the tenant is paid back, and use Create PDF Receipt to produce a clear document for records.
