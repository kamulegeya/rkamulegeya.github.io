1) Add Property Owner

Menu path: Manage Properties → Property Owners → Add (+)

Steps

  1. Open Manage Properties (Property Management).
  2. Open the Property Owners tab.
  3. Tap Add (or +) to open Add new Property Owner.
  4. Complete owner details and commission settings, then tap Add new Property Owner.

Owner details fields

Field (as shown in the app) What to enter Notes
Enter Name Owner’s full name (person) or legal company name. Use the name you want shown on documents.
Enter Tel. Number Owner’s phone number. Use a reachable number.
Enter Physical Address Owner’s address. Use a billing-friendly format.
Enter Email Owner’s email address. Recommended for statements/communication where used.
Enter TIN Tax Identification Number. Required where tax compliance applies.
Enter Mobile Money / Bank Information Payment instructions (bank + mobile money details). Critical: printed on invoices/receipts. Make it detailed and accurate.
VAT Registered (Yes/No) Select Yes if VAT registered; otherwise No. For commercial owners where applicable. Commission calculations are net of VAT where applicable.
Enter Home Country Owner’s home country. Useful for record completeness/reporting.

Commission Settings

Commission can be Fixed, Percent, or Hybrid (Fixed + Percent).

  • Fixed: calculated by unit:
    • OWNER_PER_MONTH
    • UNIT_PER_MONTH
    • OCCUPIED_UNIT_PER_MONTH
  • Percent: percentage of RECEIPT (net of VAT where applicable) or INVOICE.
  • Hybrid: Fixed fee (using one of the units above) + Percent of RECEIPT or INVOICE.
Logo for printing: Use Add / Update Log(o) to attach the owner’s logo for invoices/receipts and other printed documents (depending on templates).

Bulk Import Property Owners (Initial Setup)

If you are onboarding an agency (or portfolio) with many owners, you can import owners in bulk using a CSV file created from a template generated inside the app. This is faster than creating owners one by one.

When to use bulk import

  • Initial setup where an agency has many property owners to register.
  • You already have owner data in Excel and want to migrate it into the app.

How it works (Template → Fill in Excel → Save CSV → Import)

  1. On Manage Properties → Property Owners, tap the template button (the button with a small “+” icon). The app generates/downloads an owners import template.
  2. Open the template on a computer (Excel or any spreadsheet tool) and fill in the owner records. Complete only the provided columns.
  3. Save/export the completed sheet as a CSV file (Comma Separated Values).
  4. Back on the Property Owners screen, tap the CSV Import button (the button labeled CSV), then select the saved CSV file to import the owners.
Important rules for successful import:
  • Do not change the template headers (column names) or their order unless the app explicitly allows it.
  • Ensure all required fields in the template are completed for every owner row.
  • Use consistent formatting for phone numbers and emails to avoid duplicates and failed imports.
  • After import, open a few owners and confirm payment instructions, VAT registration, and commission settings are correct.

2) Add Property (Apartment Block)

Menu path: Manage Properties → View Properties → Add (+)

Prerequisites

  • Create at least one Property Owner.
  • Create at least one Location.

Fields (spinners + text)

Field What to select / enter Notes
Property Owner (spinner) Select the owner. Links the property to an owner for reporting and statements.
Location (spinner) Select the location. Used for grouping and reporting.
Property Name (text field) Enter the property name. Use a clear, unique name (e.g., “Nansana Rentals”).
Property Type (spinner) Select Residential, Commercial, or Hybrid. Hybrid = mixed residential + commercial units.

5) Add Usage Type (Apartment Usage & VAT)

Menu path: Manage Properties → Use Types → Add (+)

Usage Types define how apartments (rental units) are used (for example: Residential or Commercial) and the VAT/tax rules that apply to that usage. This matters most when the usage is Commercial and the Property Owner is marked as VAT Registered = Yes.

Field (as shown in the app) What to enter / select Notes
Usage Name (e.g. Commercial) Enter a short usage name. Examples: Commercial, Residential, Exempt (if used).
VAT Applicable (spinner) Select the VAT applicability option (e.g., Standard). Choose what matches your tax rules and business policy.
VAT Rate (%) Enter the VAT rate as a number (e.g., 18). Do not include the % sign.
VAT Inclusive (toggle) ON if amounts already include VAT; OFF if VAT should be added on top. Ensure this matches how you price rent and charge billable items.
Practical rule: VAT handling is driven by the unit’s Usage Type and the Property Owner VAT Registered setting. For example, Commercial usage + VAT-registered owner typically requires VAT settings (rate and inclusive/exclusive) to be applied correctly.

6) Add Rental Unit (Apartment / Unit)

Menu path: Manage Properties → View Apartments → Add (+)

Apartment list actions: On the View Apartments screen, long-press an apartment card (or use the ⋮ menu) to open the context menu with actions like Delete, Discontinue Apartment, Apartment out of Use, Make Apartment Available, and Edit.

Prerequisites

  • Property created (View Properties).
  • Rental Type created (Rental Types).
  • Currency created (Currency).
  • Usage Type created (Use Types).

Unit details fields

Field (as shown in the app) What to select / enter Notes
Property (spinner) Select the property/apartment block. Links unit to the correct property.
Rental Type (spinner) Select the unit category (e.g. Two-Bedroom Apartments). Managed under Rental Types.
Currency (spinner) Select billing currency (e.g. Uganda Shillings). Shown on invoices/receipts for this unit.
Use Type (spinner) Select apartment usage/VAT type (e.g. Residential (Exempt from VAT)). Controls VAT treatment and inclusive/exclusive behavior for this unit.
Unit Name (text field) Enter unit identifier (e.g. House 1, Room B2, Shop 03). Keep unique within the property.
Monthly Rental Amount (text field) Enter monthly rent amount. Use numbers only (example: 500000) unless the app allows separators.

Apartment list (long-press) context menu actions

Action When to use it What it does / important rules
Delete Only when the unit was created by mistake (e.g., duplicate/wrong entry). Permanently removes the unit only if no tenancy record exists for that unit. If a tenancy has ever existed, use Apartment out of Use or Discontinue Apartment instead to preserve history.
Discontinue Apartment When the unit is permanently no longer lettable. Marks the unit as discontinued (keeps historical records). Typical cases: space converted into other uses, a house remodelled into a different layout, or a residential unit remodelled into commercial office space.
Apartment out of Use When the unit is temporarily unavailable. Marks the unit as not available for letting for a period (keeps records), e.g., under maintenance, repairs, or short-term renovation. Use Make Apartment Available when it is ready to let again.
Make Apartment Available After a unit was set to Apartment out of Use. Returns the unit to an available/lettable state so it can be allocated again.
Edit When you need to correct or update unit details. Updates unit attributes such as unit name/number, rental type, rent amount, currency, and use type.

Bulk Import Rental Units (Initial Setup)

If you are setting up a property with many units (e.g., dozens or hundreds), you can import units in bulk instead of adding them one by one. This is done using a CSV file created from a template generated inside the app.

When to use bulk import

  • Initial onboarding where a substantial number of rental units must be captured quickly.
  • Data already exists in Excel and needs to be migrated into the app using the provided template.

How it works (Template → Fill in Excel → Save CSV → Import)

  1. On Manage Properties → View Apartments, tap the template button (the button with a small “+” icon). This generates/downloads an import template.
  2. Open the template on a computer (e.g., laptop) using Excel (or any spreadsheet tool). Fill in unit details in the provided columns only.
  3. Save/export the completed sheet as a CSV file (Comma Separated Values).
  4. Back on the View Apartments screen, tap the CSV Import button (the button labeled CSV). Select the saved CSV file to import the units.
Important rules for successful import:
  • Do not change the template headers (column names) or their order unless the app explicitly allows it.
  • Ensure required fields in the template are completed for every row (unit).
  • Use valid values that already exist in the app (e.g., Property, Rental Type, Currency, Use Type), exactly as shown in the system lists.
  • Prefer saving CSV as UTF-8 if your unit names include special characters.
  • After importing, review a few units on the list to confirm amounts, currency, and use type are correct.
Recommended practice: Once a unit has any tenancy history, avoid Delete. Use Apartment out of Use for temporary unavailability and Discontinue Apartment for permanent retirement, so reporting and audit trail remain intact. Also use Show status history on the unit card to review status changes over time.

7) Add Tenancy Term

Menu path: Manage Properties → Tenancy Terms → Add (+)

Tenancy Terms define billing rules such as billing cycle, billing mode (calendar vs anniversary), first invoice logic, proration, grace days, deposit, escalation, and late payment charges.

Field (as shown in the app) What to enter / select Notes / Examples
Billing cycle (spinner: e.g., Monthly) Select how often rent is billed (Monthly, Quarterly, Half-yearly, Annual). This is the billing frequency used to generate invoices.
Billing Mode (spinner) Select how billing periods are anchored. Calendar aligned: Use this billing mode if you want invoices to follow normal calendar boundaries (e.g., 1st to month-end), regardless of when the tenant moved in. This applies when tenants join mid-month. You must decide how to treat the partial period: either bill it separately (prorated) and then bill N full periods, or count the partial period within the initial N periods.

Anniversary aligned: Use this billing mode if you want invoices to follow the tenant’s start date rather than the normal calendar. For example, a tenant who enters on the 12th will have invoices starting on the 12th and ending on the 11th of the next month (for a monthly cycle).
First Invoice Rule (spinner) Select how the first invoice period is calculated. Partial period + N full periods (calendar-aligned): bills move-in partial period to the next boundary (e.g., month-end), then bills N full periods.
N full periods from start date (anniversary-aligned): bills N full periods starting from the tenancy start date.
Prorate partial periods (move-in/move-out) (checkbox) Tick to charge a prorated amount for partial periods. Recommended for calendar-aligned billing when a tenancy starts mid-period. When anniversary aligned is selected, this option is typically hidden/unused.
Proration method (spinner) Select how proration is calculated. Actual days in period: uses the actual number of days in the calendar period (recommended for residential).
30-day convention: uses a 30-day month convention (commonly used in some commercial settings).
Term Name Enter a clear name. Example: “Monthly – Calendar aligned – 5 Grace Days”.
Rent Grace Days Number of grace days after due date. Example: 5.
Security Deposit Fee Deposit amount required at move-in. Use 0 if no deposit is required.
Initial Payment Periods Number of billing periods billed on the first invoice. Example: 3 (for a 3-month initial invoice on a Monthly cycle). For non-monthly cycles, this is still “number of periods”, not “months”.
Escalation Rate (e.g., 0.05 for 5%) Enter escalation rate as a decimal. Important: 0.05 = 5% (do not enter 5).
Escalation Frequency (months) How often escalation applies, in months. Example: 12 = yearly escalation.
Apply Late Payment Charges (checkbox) Tick if late payment charges apply. Enable if your agreements include penalties.
Tip: If you choose Calendar aligned and the tenant moves in mid-month, the first invoice can include a partial period plus the configured number of full periods. If proration is enabled, the partial period is billed using the selected proration method.

8) Add Billable Item

Menu path: Manage Properties → Billable Items → Add (+)

Billable Items are additional charges besides rent (e.g., Cleaning Service, Maintenance Fee, Late Payment Charge). When billing, VAT treatment depends on: (1) whether the item is marked Is Vatable?, (2) the unit’s Usage Type (e.g., Commercial vs Residential and VAT settings), and (3) whether the Property Owner is VAT Registered = Yes.

Field (as shown in the app) What to enter / select Notes
Item Name Name of the charge. Example: Cleaning Service, Garbage Collection, Maintenance Fee.
Is Vatable? (checkbox) Tick if this item should attract VAT where applicable. If unticked, the item is treated as Non-VATable. If ticked, VAT may apply based on the unit’s Usage Type and owner VAT registration.
Tip: Keep billable item names consistent to avoid duplicates (e.g., use either “Cleaning Service” or “Cleaning”, not both).
